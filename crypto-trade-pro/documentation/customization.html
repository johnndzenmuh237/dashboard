Customization

Every visual decision in CryptoTradePro is driven by CSS custom properties (variables), so most rebranding can be done by editing a handful of values in one place rather than hunting through every file.

1. Color tokens

All core colors live at the top of assets/css/style.css inside the :root block. Changing a value here updates it everywhere the variable is used.

:root {
  --bg-base:        #0B0E11;  /* page background */
  --bg-panel:       #161A1E;  /* card / sidebar background */
  --bg-elevated:    #1E2329;  /* inputs, table hover, chips */

  --accent-green:   #0ECB81;  /* buy / positive / primary actions */
  --accent-red:     #F6465D;  /* sell / negative */
  --accent-amber:   #F0B90B;  /* highlights, staking, warnings */

  --text-primary:   #EAECEF;
  --text-secondary: #848E9C;
  --text-tertiary:  #5E6673;
}

To rebrand the whole app to a new primary color, change --accent-green and --accent-green-dim — every button, active nav link, and positive-value indicator updates automatically.

2. Switching themes at runtime

Two independent systems control appearance, both defined in assets/css/darkmode.css and assets/css/themes.css:

  • Light / dark mode — toggled by setting data-theme="dark" or data-theme="light" on the <html> element. The toggle button with class .theme-toggle-btn (in the top bar and in Settings → Preferences) does this automatically via assets/js/app.js.
  • Accent palette — toggled by setting data-accent to classic, ocean, violet, or terminal. The swatch picker on the Settings page (.theme-swatch elements) drives this.

Both choices are saved to localStorage so they persist across page loads and across the whole site.

3. Adding a new accent theme

Open assets/css/themes.css and copy one of the existing blocks, for example:

html[data-accent="myBrand"] {
  --accent-green:   #YOUR_PRIMARY_COLOR;
  --accent-green-dim: rgba(R, G, B, 0.12);
  --accent-red:     #YOUR_NEGATIVE_COLOR;
  --accent-red-dim: rgba(R, G, B, 0.12);
  --accent-amber:   #YOUR_HIGHLIGHT_COLOR;
  --accent-amber-dim: rgba(R, G, B, 0.12);
}

Then add a matching swatch button in settings.html using the existing .theme-swatch markup, with data-swatch="myBrand".

4. Typography

Two typefaces are loaded from Google Fonts in every page's <head>: Inter for interface text and JetBrains Mono for all numeric data (prices, balances, percentages). Both are referenced via CSS variables:

--font-ui:   'Inter', -apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif;
--font-mono: 'JetBrains Mono', 'SFMono-Regular', Consolas, monospace;

To swap fonts, update the Google Fonts <link> tag in each page's head and change these two variables in style.css.

5. Logo and branding

The sidebar and auth-page logos reference assets/images/logos/logo-mark.svg and the browser tab icon references assets/images/logos/favicon.png. Replace these two files with your own (same filenames) to rebrand instantly across all 14 pages — no HTML edits required.

6. Replacing placeholder market data

All coin names, prices, and percentages come from a single object in assets/js/app.js:

const MarketData = {
  coins: [
    { id: 'btc', name: 'Bitcoin', ticker: 'BTC', price: 67342.18, change24h: 2.34, ... },
    ...
  ]
};

Every page (dashboard, markets, portfolio, trading, staking) reads from this list. Edit the array to change which assets appear, or connect a real price feed by replacing the static array with a fetch call — the rendering code in each page already expects this exact shape, so no other markup changes would be needed.

If you connect this template to a live price feed, wallet provider, or exchange API, have that integration security-reviewed separately — this template's JavaScript was written for front-end demonstration only and does not implement authentication, key management, or transaction signing.

7. Layout spacing and breakpoints

Sidebar width, top bar height, and ticker height are all variables, so resizing the shell is a one-line change in style.css:

--sidebar-width: 232px;
--topbar-height: 64px;
--ticker-height: 40px;

Responsive breakpoints (1200px, 1024px, 768px, 480px) live in assets/css/responsive.css, organized from largest to smallest screen.
