Assets guide

This template ships without bundled photography, coin icons, or avatars — only code. Add files with the exact names below to the matching folder under assets/images/ and every page will pick them up automatically.

1. Logos — assets/images/logos/

FilenameUsed onRecommended size
logo-mark.svgSidebar brand, login, register, forgot-passwordSquare, scales cleanly at 26–30px
favicon.pngBrowser tab icon, all pages512×512px PNG

2. Coin icons — assets/images/crypto/

Referenced by ticker symbol from the shared MarketData object in app.js. Used in the ticker strip, market tables, portfolio holdings, wallets, staking, and the trading pair selector.

Recommended: 64×64px PNG, transparent background, circular crop (the CSS already applies border-radius: 50%, so a square source image works fine).

3. Avatars — assets/images/avatars/

The profile page also supports live avatar uploads via the camera icon (handled client-side in settings.js) — that preview doesn't require a file on disk, but the default image shown before upload does.

Recommended: 600×600px JPG or WebP, square aspect ratio (cards render at aspect-ratio: 1).

5. Icons — assets/images/icons/

This folder is reserved for any custom icon assets you want to add beyond the inline SVGs already used throughout the template (navigation, buttons, status indicators). All built-in icons are hand-coded inline SVG with stroke="currentColor", so they inherit theme colors automatically and need no files here by default.

Missing image files will show as broken-image icons in the browser rather than breaking layout — every <img> tag has defined dimensions or sits inside a sized container, so the page structure stays intact while you add assets incrementally.
